# 週末フットサルの目的

この場所は、趣味のフットサルを続けていく中で、自分の目的を確認し、振り返り、また立ち戻るための場所です。

記録用の簡単なWebアプリは [index.html](index.html) にある。ブラウザで開くと、身体と心の状態を0から10で記録し、からだ、ことば、いしき、トライ、振り返りを保存できる。

記録日時は年月日時分まで残せる。同じ日に複数回記録する場合も、タイミングを分けて振り返れる。

## 自分の目的

週末フットサルの目的は、最後まで走り切れる自分でありながら、挑戦したいプレーにトライし、ゴールやアシストで仲間に貢献することで、清々しい達成感と向上心を育てること。

短く言うと、

> 挑戦し、走り切り、仲間に貢献して、清々しい向上心を持ち帰ること。

## 大切にしたい要素

- 挑戦: やってみようと思ったプレーに実際にトライする
- 貢献: ゴールだけでなく、アシストや味方を活かすパスで役に立つ
- 走力: 最後の時間まで走って、試合の流れについていく
- 成長感: 終わったあとに「少し前に進めた」と思える
- 向上心: 半年後にも残っていてほしい、自分の核になる感覚
- 安定感: 失敗しても大きく構えて、次のトライに向かう

## 本番で安定するための準備

本番で安定した力を出すには、技術だけでなく、価値観、意識、技術の3つを準備する。

1. 価値観: 自分はどうありたいか
2. 意識: 今日は何に意識を向けるか
3. 技術: 具体的に何をトライするか

まず価値観を大きく持つ。

> 自分は、失敗してもいいから、挑戦し、走り切り、仲間に貢献する人でありたい。

味方から「あそこのシーンはもったいなかった」と言われる場面は、トライできる余地があった場面でもある。そこで小さくならず、価値観を土台にして大きく構える。その上で、次はトライする。

詳しくは [docs/preparation.md](docs/preparation.md) にまとめている。

考えすぎて動けなくなるときは、マインドを体感覚に落とし込む。

例:

- 大きく構える: 胸を開く、目線を上げる、呼吸を吐く
- トライする: 最初の一歩を出す、パスコースを見る、足を止めない
- 切り替える: 顔を上げる、戻る方向へ身体を向ける

詳しくは [docs/body-sense.md](docs/body-sense.md) にまとめている。

結果は直接コントロールしにくい。だから、結果を最初に考えすぎるより、コントロールしやすいものから整える。

1. からだ: 姿勢、呼吸、目線、最初の一歩
2. ことば: 自分にかける短い言葉
3. いしき: 何を見るか、何へ注意を向けるか

詳しくは [docs/mental-control.md](docs/mental-control.md) にまとめている。

整えた後は、もう一度身体と心に点数をつける。試す前と後の変化を比べることで、自分で状態を動かせる感覚を確認する。点数は本人の主観でつけ、周りの人は余計な口出しをしない。

## 来てよかったと思える状態

- 清々しい気持ちで終われた
- やりきった感覚がある
- トライしてみようと思ったことに、実際にトライできた
- 味方からのアシストをうまくゴールに決められた
- 味方にもアシストパスを出せた
- 「本郷さん、サンキュー」と言われるような貢献ができた
- 最後の時間まで走り、試合の流れについていけた

## 目的論と原因論

フットサルを振り返るときは、原因論と目的論の両方を使う。

原因論は「なぜそうなったのか」を考えること。できなかった理由、体力が落ちた場面、判断が遅れた原因を見つける。

目的論は「ありたい姿に向かうためにどうするか」を考えること。次に何をトライするか、どんな自分でプレーしたいかを決める。

原因を見つけるだけで終わらせず、次の目的に変える。

## 週末に行く前の一言

毎回、行く前に1つだけ決める。

例:

- 今日は裏へのパスを1本狙う
- 今日はシュートを迷わず打つ
- 今日は最後の5分でも戻り切る
- 今日は味方のゴール前への動きを1回見逃さない
- 今日は味方に「ありがとう」と言われるプレーを1つ作る

## 振り返りの使い方

毎回完璧に書く必要はない。短くてもいいので、次の3つだけ残す。

1. 今日トライしたこと
2. できたこと、できなかったこと
3. 次にトライすること

記録用のテンプレートは [templates/reflection.md](templates/reflection.md) に置いている。

ブラウザで記録する場合は [index.html](index.html) を開く。記録はブラウザ内に保存され、MarkdownまたはJSONで書き出せる。

Webアプリは、記録入力、記録一覧、記録詳細、Docs一覧、Docs詳細の画面に分けている。記録詳細は一覧から開ける。Docsでは、このリポジトリ内のMarkdown文書をアプリ内で読み返せる。

入力途中の内容は「途中保存」で下書き保存できる。入力中も自動で下書き保存され、同じブラウザで開き直すと復元される。「記録する」を押すと履歴に確定保存され、下書きは消える。

目的そのものは [docs/purpose.md](docs/purpose.md)、考え方は [docs/thinking.md](docs/thinking.md)、本番前の準備力は [docs/preparation.md](docs/preparation.md)、意識系の理解は [docs/awareness.md](docs/awareness.md)、体感覚への落とし込みは [docs/body-sense.md](docs/body-sense.md)、メンタルの整え方は [docs/mental-control.md](docs/mental-control.md)、狭いコートでのプレッシャー対処は [docs/pressure-control.md](docs/pressure-control.md) に分けている。

## GitHub Pages

このアプリは静的HTML/CSS/JavaScriptなので、GitHub Pagesで公開できる。

公開方法:

1. GitHubでリポジトリを作る
2. このフォルダを `main` ブランチとしてpushする
3. GitHubの `Settings > Pages` で `Build and deployment` の `Source` を `GitHub Actions` にする
4. `Deploy GitHub Pages` ワークフローが完了したら公開URLにアクセスする
